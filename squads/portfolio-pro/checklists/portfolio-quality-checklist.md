# Portfolio Quality Checklist

**Checklist ID:** portfolio-quality
**Version:** 1.0
**Purpose:** Validate a professional portfolio against quality standards before delivery
**When to use:** After all 5 phases of wf-portfolio-pipeline are complete

---

## Phase 1: Brand Positioning (Tier 0)

### Clarity
- [ ] Core brand statement defined (1 sentence, under 15 words)
- [ ] 3 differentiators identified and documented
- [ ] Target audience clearly defined with specific attributes
- [ ] "What I want to be known for" statement articulated

### Consistency
- [ ] Portfolio tone matches brand positioning
- [ ] Visual identity aligns across all sections
- [ ] Messaging consistent between bio, case studies, and CTA
- [ ] Cross-channel consistency plan documented (LinkedIn, site, etc.)

### Assessment
- [ ] Gap analysis between current perception and desired brand completed
- [ ] Action items for closing gaps documented

**Phase 1 Gate:** All Clarity items MUST pass. Consistency items SHOULD pass.

---

## Phase 2: Narrative (Tier 1)

### StoryBrand Elements
- [ ] BrandScript complete with all 7 SB7 elements
- [ ] Client/viewer positioned as the HERO (not the portfolio owner)
- [ ] Portfolio owner positioned as the GUIDE with empathy + authority
- [ ] Clear PLAN communicated (how to engage/hire)
- [ ] SUCCESS vision painted (what happens when they hire you)
- [ ] FAILURE stakes communicated (what they miss without you)

### Copy Elements
- [ ] One-liner/tagline crafted (Problem + Solution + Result)
- [ ] Short bio (2-3 sentences) written
- [ ] Long bio (2-3 paragraphs) written
- [ ] Section headlines follow narrative arc
- [ ] CTA copy is action-oriented and clear

**Phase 2 Gate:** All StoryBrand items MUST pass.

---

## Phase 3: Design System (Tier 2)

### Typography
- [ ] Heading typeface selected with rationale
- [ ] Body typeface selected with rationale
- [ ] Type scale defined (h1 through body, captions)
- [ ] Line height and spacing system documented

### Layout
- [ ] Grid system defined (columns, margins, gutters)
- [ ] Responsive breakpoints specified (if web)
- [ ] White space rules documented
- [ ] Component spacing system defined

### Visual Identity
- [ ] Color palette defined (primary, secondary, accent, neutrals)
- [ ] Color usage guidelines documented
- [ ] Photography/imagery style defined
- [ ] Icon/illustration style defined (if applicable)

### Visual Storytelling
- [ ] Before/After structure applied to case studies
- [ ] Emotional progression mapped across portfolio flow
- [ ] Data visualization guidelines defined
- [ ] Master page/slide templates created (minimum 3)

**Phase 3 Gate:** Typography and Layout items MUST pass. Visual Storytelling SHOULD pass.

---

## Phase 4: Case Studies (Tier 1)

### Structure (per case study)
- [ ] Context section (client, role, timeline, team size)
- [ ] Problem/Challenge clearly articulated
- [ ] Process documented (approach, methodology, tools)
- [ ] Solution presented with supporting visuals
- [ ] Results quantified with metrics where possible

### Quality
- [ ] 3-5 case studies included (not fewer, not more than 6)
- [ ] Each case study scannable in under 2 minutes
- [ ] Diversity of skills demonstrated across cases
- [ ] Strongest case study positioned first
- [ ] "What I Learned" included in at least 2 cases
- [ ] Thumbnail summaries created for overview page

### Writing
- [ ] Written in first person (I/we)
- [ ] Plain language (no unnecessary jargon)
- [ ] Impact statements lead each case study
- [ ] Written BEFORE designed (content drove layout)

**Phase 4 Gate:** All Structure items MUST pass for each case study.

---

## Phase 5: Sharing Strategy (Tier 2)

### Channels
- [ ] Primary channel defined (LinkedIn, personal site, Behance)
- [ ] At least 3 channels planned
- [ ] Content adaptation strategy per channel

### Content Plan
- [ ] 30-day content calendar created
- [ ] Process content ideas documented (behind-the-scenes)
- [ ] Teaching content ideas documented (share expertise)
- [ ] Remix strategy for each case study

### Anti-Spam
- [ ] Value-to-ask ratio planned (minimum 9:1)
- [ ] Community engagement approach defined
- [ ] Portfolio URL strategy finalized

**Phase 5 Gate:** Channels items MUST pass. Content Plan SHOULD pass.
